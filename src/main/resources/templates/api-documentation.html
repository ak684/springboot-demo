Impact Foresight API Documentation

Welcome to the Impact Foresight API. This API allows you to programmatically access aggregated impact indicators for your portfolio.

Key Features:

  • 📊 Access all your portfolio's aggregated indicators
  • 🌐 Multi-language support (English and Slovak)
  • 🔢 Flexible number formatting (EU/US)
  • 📈 Optional live counter data for real-time displays NEW

Authentication

All API requests require authentication using an API key. Include your API key in the request headers:

X-API-Key: YOUR_API_KEY
Note: Contact Impact Foresight to obtain your API key for your portfolio.

Base URL

https://app.impactforesight.io

Endpoints

Get Aggregated Indicators

GET/api/v1/public/portfolios/{portfolioId}/indicators

Retrieves all aggregated indicators for a specific portfolio.

Parameters

Parameter Type Description
portfolioId number The ID of your portfolio

Response Fields

Field Type Description
indicator_id number Unique identifier for the indicator
category string Category of the indicator (e.g., Environmental, Social)
indicator_name_en string Indicator name in English
indicator_name_sk string Indicator name in Slovak
indicator_value number The calculated value of the indicator
time_period string Time period code (ytd, mtd, currentYearFull, etc.)
time_period_en string Custom English time period label (e.g., "Q1 2024", "Daily Progress")
time_period_sk string Custom Slovak time period label (e.g., "Q1 2024", "Denný pokrok")
unit string Unit of measurement
unit_sk string Unit of measurement in Slovak
decimal_places number Number of decimal places to display
number_format string Number format (EU or US)
calculation_type string "prorated" for YTD/MTD calculations, "full" otherwise
hover1_*, hover2_*, hover3_* various Optional hover indicators with same field structure
continuous_counter object Optional - Present for YTD/MTD/today indicators with live counter enabled (see Continuous Counter section)

Hover Indicator Fields

Each main indicator can have up to 3 hover indicators (hover1, hover2, hover3) with the following fields:

Field Type Description
hover1_name, hover2_name, hover3_name string Name of the hover indicator
hover1_name_sk, hover2_name_sk, hover3_name_sk string Name of the hover indicator in Slovak
hover1_value, hover2_value, hover3_value number Calculated value of the hover indicator
hover1_time_period, hover2_time_period, hover3_time_period string Time period code for the hover indicator
hover1_time_period_en, hover2_time_period_en, hover3_time_period_en string Custom English time period label for the hover indicator
hover1_time_period_sk, hover2_time_period_sk, hover3_time_period_sk string Custom Slovak time period label for the hover indicator
hover1_unit, hover2_unit, hover3_unit string Unit of measurement for the hover indicator
hover1_unit_sk, hover2_unit_sk, hover3_unit_sk string Unit of measurement in Slovak for the hover indicator
hover1_decimal_places, hover2_decimal_places, hover3_decimal_places number Number of decimal places to display
hover1_number_format, hover2_number_format, hover3_number_format string Number format ("US" or "EU")

Code Examples

JavaScript (Fetch API)

const API_KEY = 'YOUR_API_KEY';
const PORTFOLIO_ID = YOUR_PORTFOLIO_ID;

fetch(`https://app.impactforesight.io/api/v1/public/portfolios/${PORTFOLIO_ID}/indicators`, {
  headers: {
    'X-API-Key': API_KEY,
    'Content-Type': 'application/json'
  }
})
.then(response => {
  if (!response.ok) {
    throw new Error(`HTTP error! status: ${response.status}`);
  }
  return response.json();
})
.then(data => {
  // Process your indicators
  data.forEach(indicator => {
    console.log(`${indicator.indicator_name_en}: ${indicator.indicator_value} ${indicator.unit}`);
    
// Check if this indicator has live counter data
    if (indicator.continuous_counter) {
      console.log(`  Live counter enabled - Rate: ${indicator.continuous_counter.rate_per_second}/sec`);
    }
    
    // Format numbers according to locale
    const formattedValue = indicator.number_format === 'EU' 
      ? indicator.indicator_value.toLocaleString('sk-SK', { 
          minimumFractionDigits: indicator.decimal_places,
          maximumFractionDigits: indicator.decimal_places 
        })
      : indicator.indicator_value.toLocaleString('en-US', { 
          minimumFractionDigits: indicator.decimal_places,
          maximumFractionDigits: indicator.decimal_places 
        });
        
    console.log(`Formatted value: ${formattedValue}`);
  });
})
.catch(error => {
  console.error('Error fetching indicators:', error);
});

Python (requests library)

import requests
import json

API_KEY = 'YOUR_API_KEY'
PORTFOLIO_ID = YOUR_PORTFOLIO_ID
URL = f'https://app.impactforesight.io/api/v1/public/portfolios/{PORTFOLIO_ID}/indicators'

headers = {
    'X-API-Key': API_KEY,
    'Content-Type': 'application/json'
}

try:
    response = requests.get(URL, headers=headers)
    response.raise_for_status()  # Raise an exception for bad status codes
    
    indicators = response.json()
    
    # Process indicators
    for indicator in indicators:
        print(f"{indicator['indicator_name_en']}: {indicator['indicator_value']} {indicator['unit']}")
        
        # Check for continuous counter
        if 'continuous_counter' in indicator:
            cc = indicator['continuous_counter']
            print(f"  📊 Live counter - Daily rate: {cc['daily_rate']:.2f} {indicator['unit']}/day")
        
        # Handle hover indicators
        for i in range(1, 4):
            hover_name = f'hover{i}_name'
            if indicator.get(hover_name):
                hover_value = indicator[f'hover{i}_value']
                hover_unit = indicator[f'hover{i}_unit']
                print(f"  - {indicator[hover_name]}: {hover_value} {hover_unit}")
                
except requests.exceptions.RequestException as e:
    print(f"Error fetching indicators: {e}")

Number Formatting

The API provides formatting hints for displaying numbers correctly:

  • EU format: Uses comma as decimal separator (1.234,56)
  • US format: Uses period as decimal separator (1,234.56)
  • decimal_places: Specifies how many decimal places to show

Continuous Counter Feature Optional

Some indicators support real-time incrementing display. When enabled, prorated indicators (YTD, MTD, today) include additional data for creating live counters.

Important: The presence of the continuous_counter object indicates the feature is active. If the indicator doesn't have live counter enabled, this entire object will be absent from the response.

Implementation Examples

Vanilla JavaScript Live Counter

// Create a live counter for an indicator
function createLiveCounter(indicator, elementId) {
  const element = document.getElementById(elementId);
  
  if (!indicator.continuous_counter) {
    // Not a continuous counter, just display the value
    element.textContent = formatNumber(indicator.indicator_value, indicator);
    return;
  }
  
  const counter = indicator.continuous_counter;
  let currentValue = indicator.indicator_value;
  const startTime = Date.now();
  
  // Update the counter every second
  const interval = setInterval(() => {
    const elapsedSeconds = (Date.now() - startTime) / 1000;
    currentValue = indicator.indicator_value + (counter.rate_per_second * elapsedSeconds);
    
    element.textContent = formatNumber(currentValue, indicator);
  }, 1000);
  
  // Return cleanup function
  return () => clearInterval(interval);
}

// Format number according to indicator settings
function formatNumber(value, indicator) {
  const locale = indicator.number_format === 'EU' ? 'de-DE' : 'en-US';
  return value.toLocaleString(locale, {
    minimumFractionDigits: indicator.decimal_places,
    maximumFractionDigits: indicator.decimal_places
  }) + ' ' + indicator.unit;
}

React Hook for Live Counter

import { useState, useEffect } from 'react';

// Custom hook for live counter
function useLiveCounter(indicator) {
  const [currentValue, setCurrentValue] = useState(indicator.indicator_value);
  
  useEffect(() => {
    if (!indicator.continuous_counter) {
      return;
    }
    
    const counter = indicator.continuous_counter;
    const startTime = Date.now();
    const startValue = indicator.indicator_value;
    
    const interval = setInterval(() => {
      const elapsedSeconds = (Date.now() - startTime) / 1000;
      const newValue = startValue + (counter.rate_per_second * elapsedSeconds);
      setCurrentValue(newValue);
    }, 1000);
    
    return () => clearInterval(interval);
  }, [indicator]);
  
  return currentValue;
}

Vue.js Composition API

// Vue 3 Composition API
import { ref, onMounted, onUnmounted, computed } from 'vue';

export function useLiveCounter(indicator) {
  const currentValue = ref(indicator.value.indicator_value);
  let interval = null;
  
  const isLive = computed(() => 
    !!indicator.value.continuous_counter
  );
  
  const formattedValue = computed(() => {
    const locale = indicator.value.number_format === 'EU' ? 'de-DE' : 'en-US';
    return currentValue.value.toLocaleString(locale, {
      minimumFractionDigits: indicator.value.decimal_places,
      maximumFractionDigits: indicator.value.decimal_places
    });
  });
  
  onMounted(() => {
    if (!isLive.value) return;
    
    const counter = indicator.value.continuous_counter;
    const startTime = Date.now();
    const startValue = indicator.value.indicator_value;
    
    interval = setInterval(() => {
      const elapsedSeconds = (Date.now() - startTime) / 1000;
      currentValue.value = startValue + (counter.rate_per_second * elapsedSeconds);
    }, 1000);
  });
  
  onUnmounted(() => {
    if (interval) clearInterval(interval);
  });
  
  return {
    currentValue,
    formattedValue,
    isLive
  };
}
Note: The continuous counter feature is optional. Indicators without this feature will not include the continuous_counter object in the response. Always check for its presence before implementing live updates.

Best Practices for Live Counters

  • ✅ Always check if continuous_counter exists before using it
  • ✅ Use requestAnimationFrame for smooth visual updates
  • ✅ Clean up intervals when components unmount
  • ✅ Consider performance - don't create too many simultaneous counters
  • ✅ Round displayed values appropriately using decimal_places
